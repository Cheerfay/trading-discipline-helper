# desktop-layout-polish

## 目标

优化首页 Web 端观感，避免页面内容像移动端窄列一样挤在中间，同时保持移动端单列体验。

本分支后来也承接了结果页交互细节和 prompt/越狱输入防护的收尾优化。

## 改动范围

- 首页顶部导航容器从窄列扩展到 `max-w-6xl`。
- 桌面端首页改为左右布局：左侧品牌心智，右侧真实输入工作台。
- 保留移动端自然单列布局。
- 输入区改为浅外层区域 + 独立白色输入框，避免按钮和标签被白卡层级吞掉。
- 精简左侧文案，保留：
  - `少犯一次大错，胜过小赚几次`
  - `给冲动交易踩一脚刹车。出手前，先过一遍理由、仓位和情绪。`
- 冷静卡分析详情展开后自动平滑滚动到详情顶部；详情底部增加右侧「收起」+ 上箭头。
- 仓位观察分析展开后自动平滑滚动到分析顶部。
- 结果页冷静卡和仓位节奏卡展示用户当时输入，历史记录列表也补充用户问了什么。
- 场景标签调整为更贴近真实交易心理，包括「涨太快，怕拿不住」这类场景。
- 首页标题尝试一行/自然换行，并去掉口号末尾句号。
- 新增输入防护：
  - 首页交易想法输入先拦截 prompt 提取 / 越狱 / 输出内部规则类请求。
  - 结果页仓位补充输入和仓位节奏输入也复用同一防护。
  - `/api/trading/generate-report` 与 `/api/trading/generate-position-card` 做后端兜底。

## 关键决策

- 不做营销 landing page，不增加图片或装饰性 hero。
- 不放静态样例卡，避免页面变成产品说明页。
- 第一屏仍以真实输入为主，用户打开即可使用。
- 左侧只建立产品心智，右侧只承载输入动作，避免两边文案重复。
- prompt 防护采用轻量规则层，先处理明显非交易输入，不把这类请求送进 LLM；后续根据测试样本再扩展，避免误伤真实交易表达。
- 输入防护提示文案保持产品语气：`这里不是聊天窗口。写下你现在真实的交易想法，哪怕只有一句。`
