# homepage-content-seo-polish

## 目标

在 `xiaomi-blue-ui-experiment` 的蓝色克制视觉基础上，试验补充首页内容完整度，让网站更像一个可被搜索、可被新用户快速理解的正式产品页。

关键前提：第一屏仍然是可直接使用的交易冷静卡输入台，不改成普通营销 landing page。

## 改动范围

- 首页首屏工具入口保持不变。
- 在首屏下方新增内容模块：
  - 检查范围：情绪、理由、仓位节奏。
  - 适用场景：涨太快怕拿不住、卖飞想买回、亏损想补、看到大涨想追、想加仓但理由说不清、说不清就是想动。
  - 使用流程：写想法 → 看冷静卡 → 看仓位节奏 → 历史记录复盘。
  - 明确边界：不预测涨跌、不判断具体标的好坏、不提供买卖/持有/目标价建议。
  - FAQ：解释产品能力、输入方式、仓位节奏卡和本地记录。
  - 轻量 footer：连接历史记录、隐私政策、服务条款。
- 首页 `head` 增加 canonical 链接。
- 首页文案接入 Paraglide i18n：
  - 英文默认路径 `/`
  - 中文路径 `/zh`
  - 顶部导航和底部 footer 增加语言切换。
  - 不根据浏览器语言自动跳转；英文首页仅在中文偏好浏览器且用户未明确选择过语言时，展示一条轻量中文切换提示。
  - 中文品牌名保留「交易冷静卡」。
  - 英文展示名暂定为 `Before You Trade`，避免 `Trade Brake` / `Trade Break` 同音歧义，也避开 TradeCheck、TradeGuard、TradeClarity 等通用撞词方向。
- 首页补充基础社交 SEO：
  - `og:title`
  - `og:description`
  - `og:url`
  - `og:image`
  - `twitter:*`
  - `hreflang` + `x-default`
- `sitemap.xml` 精简为当前已准备好的公开页面：
  - 首页
  - 隐私政策
  - 服务条款
  - 暂不列出 pricing/blog。
- 同步更新项目文档和上一个蓝色 UI 分支文档。

## 暂不做

- 不接入模板里的 Pricing、Blog、SupportWidget。
- 不增加账号入口和权益说明。
- 不改结果页和历史页布局。
- 不改变交易生成、仓位节奏卡和 prompt 逻辑。
- 结果页已接入双语 UI：
  - 顶部、主卡固定标签、分析详情、仓位节奏入口、删除弹窗、仓位卡固定标签都会按当前语言展示。
  - 首页新生成请求会携带当前 locale，新生成的冷静卡和仓位节奏卡会要求模型按当前语言输出。
  - 旧卡片的模型生成正文不会自动翻译；如果旧卡是在中文下生成的，切到英文后正文仍可能是中文。
- 历史页已接入双语 UI：
  - 标题、空状态、数量、场景标签、仓位前缀、删除弹窗和 SEO title 会按当前语言展示。
  - 历史卡片中的模型生成正文仍保持当时生成语言，不做自动翻译。
- 暂不重写隐私政策/服务条款正文，本分支只把入口和 sitemap 对齐到当前产品。
- 暂不处理新仓库迁移；等本分支合并 main 后，单独整理根目录仓库结构。

## 待确认

- 首屏下方内容是否增强信任感，而不是显得像泛 SaaS 页面。
- 移动端滚动长度是否可接受。
- FAQ 是否需要更偏 SEO 关键词，还是保持产品语气优先。
- 英文首页文案是否符合产品气质，尤其 `Before You Trade` 是否作为英文展示名保留。
- 法律页仍是模板通用文本，后续上线前应改成交易冷静卡专属版本。
- 若效果可接受，再考虑合并回 `main`；如果页面过重，可以保留检查范围和 FAQ，删减场景/流程模块。
