# xiaomi-blue-ui-experiment

## 目标

在 `desktop-card-layout`/`desktop-layout-polish` 的布局基础上，试验并确定一版更有产品记忆点的蓝色克制视觉方向。

关键词：清冷、克制、专业、不是炒股软件、不是普通 AI 聊天工具。

## 改动范围

- 首页、冷静卡结果页、历史页统一使用蓝色渐变背景和白色玻璃卡片。
- 首页左侧 hero 保留核心文案：
  - `少犯一次大错，胜过小赚几次`
  - `给冲动交易踩一脚刹车。出手前，先过一遍理由、仓位和情绪。`
- 首页输入区调整：
  - 输入框接近白色纸面，降低底纹存在感。
  - placeholder 使用轻蓝灰，避免过重。
  - 场景 chip 改为更明显的蓝色可点击胶囊。
  - 底部合规说明增加 `Info` 提示符号。
- 冷静卡结果页与仓位节奏卡：
  - 分析免责声明增加 `Info` 提示符号。
  - 仓位输入提示增加 `Info` 提示符号。
  - 保持冷静卡主内容居中，仓位节奏卡作为下方深入一层。
  - 移动端结果页调整为 logo、冷静卡标题、场景标签分区展示，避免顶部信息挤在一起。
  - 冷静卡和仓位节奏卡的自查问题列表在移动端统一左对齐，减少右侧参差感。
- 历史页复用同一套视觉语言。
- 新增并接入品牌 icon：
  - 正式采用「硬币轮廓 + 暂停符号」方案。
  - 更新 `public/favicon.svg` 与 `public/logo.svg`。
  - 首页、冷静卡页、历史页导航展示 logo。
- 修复移动端提交后无响应问题：
  - `/api/trading/generate-report` 与 `/api/trading/generate-position-card` 从 `createServerFn` 调用改为普通 server route handlers。
  - 避免生产预览出现 `Server function info not found`。

## 关键决策

- 不采用趋势线 icon，避免被理解为“不涨”或暗示行情走势。
- 最终选择「硬币轮廓 + 暂停符号」icon：更贴近钱和交易动作，同时强调先暂停，不暗示涨跌方向。
- 不使用红绿涨跌色，不使用 K 线，不使用过重交易终端风。
- 不增加营销模块，第一屏仍以真实输入为核心。
- 蓝色方向参考高端汽车官网的清爽感，但只吸收色调和空气感，不引入汽车元素。

## 验证

- 已验证生产构建通过：`corepack pnpm build`。
- 本地生产预览曾使用：`http://192.168.22.0:4174/`（需按当前局域网 IP 重启确认）。

## 合并前待确认

- 人工看一遍移动端首页、结果页、历史页。
- 确认品牌 icon 在浏览器 favicon、导航小尺寸和移动端顶部都清晰。
- 若确认主线采用这版视觉，再合并回 `main`。
