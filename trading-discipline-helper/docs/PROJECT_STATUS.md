## 0. 一句话

「交易冷静卡」——用户在准备买卖股票、情绪上头时打开的工具。它**不预测涨跌、不荐股、不给买卖建议**，只帮用户在出手前看清：这笔操作的理由够不够、仓位稳不稳、是不是情绪在替自己做决定。本质是**情绪服务 + 非理性交易行为风险管理 + 仓位风险管理**。

口号：**少犯一次大错，胜过小赚几次。**

## 1. 产品定位与目标用户

### 核心原则
1. 不是 AI 荐股工具，不预测涨跌，不给买卖建议。
2. 目标不是生成长报告，而是让用户**先停下来**。
3. 结果首屏是「冷静锚点」（一句话 + 一个动作），不是分析报告。
4. 先共情，再提醒，再给一个很小的行动。详细分析保留但默认折叠。
5. 视觉安静、克制、可信，不像炒股软件，不用刺激的红绿涨跌色。
6. 不审判用户，像一个理性但克制的伙伴。

### 目标用户（决定文案与内容气质）
- **画像一：有资金量但纪律差的中产散户**（20万~300万可投资产）。已意识到"输在纪律不在认知"，愿为"少犯一次大错"付费。痛点：下单上头、买入理由不清、仓位失控、盈利拿不住、亏损不止损、事后后悔。
- **画像二：长期投资者，易被短期波动干扰**。关注基本面/估值，但大涨大跌时动摇，分不清"价格波动"和"投资逻辑变化"。
- **画像三：家庭资产管理者**。对亏损敏感，要决策稳健、可解释、可记录。
- **非目标用户**（MVP 不服务）：完全小白、纯短线高频、只想暴富的人、专业机构。

气质结论：用户是**吃过亏、有认知、输在纪律的老手**，不是需要被哄的小白。文案要**冷静、专业、克制**，像一个比你更有纪律的交易伙伴——不哄你、不骂你，平静地帮你把事看清楚。

## 2. ⚠️ 合规红线（最重要，任何模型/功能都必须严守）

法规依据：《证券投资顾问业务暂行规定》（证监会公告 2020-20 号）。提供"涉及具体证券的投资建议（品种/组合/买卖时机/仓位）"且"获取经济利益"= 需持牌。第 32 条：不得通过网络等媒体作出买入/卖出/持有具体证券的建议。监管范畴已覆盖 软件/APP/社交工具。**免责声明挡不住实质认定——监管看你实质做了什么。**

### 核心分界线
> 针对"这只具体的票"给结论/仓位数字/择时 = **危险（投资建议，禁止）**
> 给"任何交易都适用的通用纪律/心理洞察" = **安全（风险教育，可做）**

### 绝不能做
- 不预测涨跌、不给买卖建议、不给目标价
- 不评价用户提到的个股好不好、值不值得买、未来怎么走
- 不给针对某只票的具体仓位数字（如"买 20%~30%""加到一半"）
- 不给择时阈值（如"突破某价位/涨跌某百分比就买/追/卖"）—— 这也算"买卖时机建议"
- 不说"建议买入/卖出""抄底""逃顶""重仓""稳赚""必涨"

### 可以做（价值所在）
- 复述用户**自己说过**的事实（标的、价格、经过、数字）——证明听懂了他
- 点破此刻的心理陷阱（如把卖出价当成"该回到的价"、用新交易补偿后悔），就事论事，不报"锚定偏差"术语
- 涉及仓位只给**通用纪律**（如"情绪上头时，任何冲动单子都该先砍到平时仓位的零头"）

### 防 AI 误用的硬规则
- **所有数字必须来自用户输入，模型一个都不许编造。** prompt 里曾因 one-shot 示例带具体股价/百分比，导致模型直接套用、误导用户——已全部清除。任何示例只示意"语气和角度"，不可照抄数字。

## 3. 当前进度

### ✅ 功能 1：交易前冷静卡（已完成）
完整用户路径已跑通：
1. **首页**：一个大输入框（写真实想法）+ 一排场景 chip（想买/想卖/想加/想割/卖飞了/追高亏了/说不清）+ 「先别急，看一眼」按钮。
2. **直接生成**（无第二页）：只要有真实内容就直接出卡；近乎空输入才前端温和提示，不调模型。
3. **结果页（冷静卡）**：首屏极简 = 状态信号灯 + headline（一段融合"安抚+点破真相"的话，首屏主角）+ 一个冷静动作。其余（3 个自查问题、完整分析、仓位补充）默认折叠。
4. **可选仓位补充**：仅当模型判断"缺仓位信息会更不准"（且为 buy/sell/add/cut 场景）时，结果页就地浮现一句邀请 + 单行输入，补完原地重新生成。
5. **历史记录（我的记录）**：localStorage 卡片流。
6. **输入防护**：首页输入、仓位补充输入、仓位节奏输入都会拦截 prompt 提取 / 越狱 / 要求输出内部规则类输入；前端先拦，后端 API 再兜底。

### ✅ 功能 2：仓位节奏卡（MVP 已完成）

已在冷静卡结果页接入第二层检查：
1. **入口**：冷静卡结果页下方「再深入一层」区域，按钮文案为「看看仓位节奏」。
2. **输入**：用户可写整体持仓，也可写这次打算动多少；提交按钮为「帮我看看仓位安排」。
3. **结果**：生成「仓位节奏卡」，只看纪律、集中度、调整节奏和现金余地，不判断具体标的该不该买卖。
4. **规则层**：仓位比例解析、单票档位、all-in、同主题集中、现金缓冲由代码规则处理；模型只把规则结果转成克制、可读的话。
5. **展示**：仓位节奏卡生成后自动平滑滚动到卡片顶部；「展开仓位观察分析」展开后自动滚到分析顶部。

### ✅ 视觉方向：蓝色克制版（已完成一轮）

当前分支已完成一轮产品气质定调：
1. **整体视觉**：从灰白克制版进一步调整为「深蓝天空渐变 + 白色玻璃工作台」方向，参考高端汽车官网的清爽蓝色，但不使用车辆/行情/红绿涨跌元素。
2. **首页**：保留左右布局，左侧建立「少犯一次大错，胜过小赚几次」心智，右侧是主输入工作台。
3. **输入区**：输入框回到接近白色纸面的感觉，placeholder 使用较轻的蓝灰色，场景 chip 改为更像可点击按钮的蓝色胶囊。
4. **结果页/历史页**：复用同一套蓝色背景、玻璃卡片、信息提示符号和品牌 icon，保证首页、冷静卡、仓位节奏卡、历史记录视觉一致。
5. **品牌 icon**：正式采用「硬币轮廓 + 暂停符号」方案，贴近钱和交易动作，同时强调先暂停、不暗示涨跌方向；已接入 `public/favicon.svg` 与 `public/logo.svg`。
6. **移动端体验**：结果页顶部信息重新排布，避免 logo、场景标签和标题挤在一起；冷静卡/仓位节奏卡问题列表在移动端统一左对齐。
7. **API 路由稳定性**：交易生成接口已从 `createServerFn` 调用改为普通 server route handlers，修复生产预览环境中移动端提交后可能出现 `Server function info not found` 的问题。

### 🚧 首页内容与 SEO 补充（当前实验中）

当前分支在不改变首屏工具入口的前提下，参考模板站点的完整信息架构，试验补充首页下方内容：
1. **检查范围**：说明交易冷静卡会检查情绪、理由、仓位节奏。

## 4. 技术栈与代码地图

技术栈：TanStack Start (Vite 8 + React 19 + TS) 模板 ShipAny Next。详见根目录 `AGENTS.md`（模板的完整说明）。

### 本产品的核心文件（功能 1 全在这几个文件里）

| 文件 | 作用 |
|---|---|
| `src/lib/trading/types.ts` | 所有类型：`Scene`、`CalmStatus`、`CalmCard`（核心对象）、`TradeCardInput`、`CalmCardRecord` |
| `src/lib/trading/llm.ts` | **统一 LLM 传输层**。`callLLM()` 按 provider 分发到 Anthropic/OpenAI/Gemini，处理各家 API 差异（endpoint/鉴权/system位置/返回结构）。`PROVIDER_DEFAULTS` 存默认模型。`normalizeBase()` 兼容中转 base-url 带不带 `/v1` |
| `src/lib/trading/claude-api.ts` | **prompt 核心**：`CALM_CARD_SYSTEM_PROMPT`（陪伴者人设 + 合规红线）+ `buildPrompt()`（拼本次上下文 + 输出 JSON schema）。文件名是历史遗留，实际是所有 provider 共用的 prompt |
| `src/lib/trading/input-guard.ts` | 输入防护：识别 prompt 提取、越狱、要求输出内部规则等非交易输入；首页、结果页和两个生成 API 共用 |
| `src/lib/trading/generate-report.ts` | 规则版 mock 生成器 `generateCalmCard()`（fallback / 离线用，真实流程走 API）。含场景化文案、评分规则、状态映射 |
| `src/lib/trading/position-rules.ts` | 仓位节奏规则层：解析用户写出的百分比，判断单票档位、all-in、同主题集中、现金缓冲和缺失信息 |
| `src/lib/trading/position-prompt.ts` | 仓位节奏卡 prompt：把规则结果交给模型转成自然、合规、克制的话术 |
| `src/lib/trading/storage.ts` | localStorage：`saveCard`/`getCards`/`getCardById`/`getCardRecords`/`deleteCard`/`updateCard`（原地更新，用于仓位补充重新生成） |
| `src/routes/index.tsx` | 首页（输入框 + chip + 直接生成 + sessionStorage 暂存草稿，浏览器返回不丢） |
| `src/routes/card/$id.tsx` | 结果页（首屏极简卡 + 折叠分析 + 可选仓位补充 + 仓位节奏卡入口与展示） |
| `src/routes/history.tsx` | 历史记录页，列表里展示用户当时问了什么，进入详情可看对应冷静卡/仓位节奏卡 |
| `src/routes/api/trading/generate-report.ts` | 服务端 API：解析 provider 配置 → 调 `callLLM` → 解析 JSON → 组装 `CalmCard`。含各种 fallback 和 needsPositionInfo 的硬保护（复盘场景永不触发） |
| `src/routes/api/trading/generate-position-card.ts` | 服务端 API：生成仓位节奏卡。先跑规则层，再调用 LLM 输出卡片文案；LLM 不可改写规则结论 |
| `src/config/index.ts` | env 配置（LLM provider/model/key/baseURL 等） |

### 数据流
```
首页输入 → POST /api/trading/generate-report → resolveLLMConfig() 选 provider
→ buildPrompt() 拼 prompt → callLLM() 调对应大模型 → 解析 JSON → CalmCard
→ 前端 saveCard() 存 localStorage → 跳转 /card/$id 展示
```

仓位节奏卡数据流：
```
结果页仓位输入 → POST /api/trading/generate-position-card
→ analyzePositionRules() 解析/分层 → buildPositionPrompt() 拼 prompt
→ callLLM() 生成卡片话术 → normalizePositionCard() 合并规则结果
→ attachPositionCard() 挂到原冷静卡 → 结果页展示
```

## 5. LLM 配置与切换（重要：换模型只改 env）

三家可切换，全靠环境变量，**不改代码**。配置在 `.env.development`（本地，已被 .gitignore，**含真实 key，不进 git**）。

```env
LLM_PROVIDER=anthropic        # anthropic | openai | gemini
LLM_MODEL=                    # 留空用默认；可手动填任意模型名（中转支持的）

# 各家 key 与 base-url（base-url 填到域名即可，代码自动拼 /v1 等路径；中转填中转地址）
ANTHROPIC_API_KEY=...
ANTHROPIC_BASE_URL=https://api.anthropic.com
OPENAI_API_KEY=...
OPENAI_BASE_URL=https://api.openai.com
GEMINI_API_KEY=...
GEMINI_BASE_URL=https://generativelanguage.googleapis.com
```

默认模型（`src/lib/trading/llm.ts` 的 `PROVIDER_DEFAULTS`）：
- anthropic = `claude-opus-4-8`
- openai = `gpt-5.5`
- gemini = `gemini-2.5-pro`

切换步骤：改 `.env.development` 的 `LLM_PROVIDER`（和对应 key）→ **重启 dev 服务**（env 启动时读取）。

### 运行
```bash
pnpm install
# 配置 .env.development（至少填好选中 provider 的 key）
pnpm dev          # http://localhost:3000
pnpm build        # 改动后务必验证构建通过
```

## 6. 待优化清单 / 已知问题

- **mock 生成器较弱**：`generate-report.ts` 是规则模板，离开真实大模型时输出很通用。它只是 fallback，真实体验依赖 LLM。
- **无账号体系**：历史记录只存当前设备浏览器的 localStorage。换浏览器/设备/清缓存即丢，多人共用一浏览器会串。MVP 阶段可接受（零注册门槛、私密感），跨设备同步/留存/付费时再上账号。首页已加"记录只保存在当前设备"说明。
- **UI 仍需移动端人工复核**：当前已完成一轮蓝色克制版视觉统一：首页、结果页、历史页都已接入同一套视觉语言。后续合并主线前建议再人工检查移动端首屏、结果页折叠详情和历史卡片密度。
- **headline 依赖输入质量**：用户写得越具体，卡越惊艳；写得模糊则退回通用安抚。这是有意为之（正向引导用户多说）。
- **prompt 是单条 user message + system**：未用强制 JSON tool 模式，靠正则提取 `{...}`，偶发解析失败已有 fallback。后续可考虑各家的 JSON mode 进一步加固。
- **输入防护仍是轻量层**：目前通过规则拦截 prompt 提取、越狱和要求输出内部规则等明显非交易输入；它是产品兜底，不是完整安全沙箱。后续可根据测试样本继续扩充，但要避免误伤真实交易表达。
- **仓位节奏卡仍是 MVP**：已能做单票/集中度/现金余地检查，但还没有反馈按钮，也没有统计用户是否真的被“刹住”。后续可以加「先不动了 / 再想想但还想操作 / 判断不太准」类反馈闭环。
- **首页内容仍在实验**：当前补充了首屏下方说明模块，需人工确认是否增强信任感，还是让页面变得过重。首屏真实输入入口不能被营销内容削弱。
- **双语范围尚未覆盖结果页**：当前只有首页完成中英文展示。结果页、历史页、模型输出、法律页正文仍需后续单独处理，尤其法律页目前仍是模板通用文本。

## 7. 需求池 / 后续优化方向

### 高优先级
- **结果反馈闭环**：在冷静卡和仓位节奏卡后增加反馈按钮，例如「先不动了」「再想想，但还想操作」「判断不太准」。这是判断产品是否真的“刹住冲动”的关键数据。
- **首页 Web 端继续打磨**：当前已从窄列改为左右布局，但还需继续观察首屏密度、移动端折叠、输入区层级和转化率。
- **历史记录信息密度**：已补“用户当时问的什么”，后续可继续增强检索/筛选/按场景回看。

### 中优先级
- **仓位节奏卡反馈与解释优化**：继续优化规则解释的柔和度，避免完整阈值表显得生硬，同时保留专业边界。
- **prompt/输入防护样本库**：收集真实测试中的 prompt 提取、越狱、误触发样本，再决定是否扩展规则。
- **文案 A/B 测试**：围绕「刹车器」心智、按钮文案、场景 chip 点击率做小步实验。

### 暂缓
- **联网搜索**：仍可作为后续能力，但不是当前 MVP 核心。即使做，也只用于理解事件背景，不做个股结论或买卖建议支撑。
- **账号和云端同步**：当前 localStorage 足够验证核心价值；跨设备、付费、长期留存阶段再上账号体系。

## 9. 给接手 AI 的快速上手指引

1. 先读本文档第 2 节（合规红线）—— 这是绝对不能踩的线，比任何功能都重要。

4. 换模型只改 `.env.development`，不改代码（见第 5 节）。
5. 任何改动后跑构建验证。本地当前用 `npm run build` 已验证通过；如果环境有 pnpm，也可按模板命令使用 `pnpm build`。
6. 做后续优化前，先读第 7 节需求池，确认当前优先级。
