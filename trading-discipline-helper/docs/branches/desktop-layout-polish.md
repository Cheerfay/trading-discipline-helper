# desktop-layout-polish

## 目标

优化首页 Web 端观感，避免页面内容像移动端窄列一样挤在中间，同时保持移动端单列体验。

## 改动范围

- 首页顶部导航容器从窄列扩展到 `max-w-6xl`。
- 桌面端首页改为左右布局：左侧品牌心智，右侧真实输入工作台。
- 保留移动端自然单列布局。
- 输入区改为浅外层区域 + 独立白色输入框，避免按钮和标签被白卡层级吞掉。
- 精简左侧文案，保留：
  - `少犯一次大错，胜过小赚几次`
  - `给冲动交易踩一脚刹车。出手前，先过一遍理由、仓位和情绪。`

## 关键决策

- 不做营销 landing page，不增加图片或装饰性 hero。
- 不放静态样例卡，避免页面变成产品说明页。
- 第一屏仍以真实输入为主，用户打开即可使用。
- 左侧只建立产品心智，右侧只承载输入动作，避免两边文案重复。

## 验证

- 已验证生产构建通过。
- 本地预览地址：`http://127.0.0.1:3000/`。

## 合并前待确认

- 需确认桌面端首屏视觉是否比窄列版本更符合产品气质。
- 移动端需要再人工看一遍，确认左右布局折叠后没有信息拥挤。
